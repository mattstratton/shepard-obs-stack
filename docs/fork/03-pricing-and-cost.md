# Spec 03 — Pricing Table & Computed-Cost Recording Rules

**Phase 3. Code-independent of specs 01–02, but end-to-end verification needs spec 02's data.**
Cost is computed in Prometheus from `shepherd_session_tokens_total` × a pricing table, via
recording rules. The pricing table is a JSON file the user edits; a generator script turns it
into a committed Prometheus rules file.

Approach rationale (decided): generated recording rules beat (b) a pushed info-metric (needs a
pusher schedule, dies with collector state) and (c) hand-edited rules (arithmetic drift).

## Files created

- `configs/pricing/model-prices.json` — the editable rate card
- `scripts/generate-pricing-rules.sh` — jq generator, stdout-first
- `configs/prometheus/rules/pricing-generated.yaml` — generated, committed, DO-NOT-EDIT header
- `configs/prometheus/rules/cost.yaml` — hand-written cost rules

## Files touched

- `configs/prometheus/prometheus.yaml` — add to `rule_files`: `'/etc/prometheus/rules/*.yaml'`
  (the whole `configs/prometheus/` dir is already volume-mounted; no docker-compose change).
  Keep recording rules OUT of `alerts/` — `tests/test-config-validate.sh` counts alerts per file
  there.

## `scripts/generate-pricing-rules.sh`

Bash + jq only (repo has no yq dependency). Behavior:

- Default: print the generated YAML to **stdout**.
- `--write`: write to `configs/prometheus/rules/pricing-generated.yaml`.
- Must be deterministic (stable ordering: file order, then type order input/output/cacheRead/cacheCreation)
  so `diff <(generate) committed-file` works as a sync test.

Generated shape — one rule per model × type:

```yaml
# GENERATED by scripts/generate-pricing-rules.sh from configs/pricing/model-prices.json — DO NOT EDIT
groups:
  - name: shepherd_model_prices
    interval: 1m
    rules:
      - record: shepherd:model_price:usd_per_mtok
        expr: label_replace(label_replace(vector(1.00), "model", "claude-haiku-4-5-20251001", "", ""), "type", "input", "", "")
      - record: shepherd:model_price:usd_per_mtok
        expr: label_replace(label_replace(vector(5.00), "model", "claude-haiku-4-5-20251001", "", ""), "type", "output", "", "")
      # ... etc
```

jq core (sketch):

```bash
jq -r '
  .prices[] as $p |
  ["input","output","cacheRead","cacheCreation"][] as $t |
  "      - record: shepherd:model_price:usd_per_mtok\n" +
  "        expr: label_replace(label_replace(vector(\($p[$t])), \"model\", \"\($p.model)\", \"\", \"\"), \"type\", \"\($t)\", \"\", \"\")"
' configs/pricing/model-prices.json
```

(plus the fixed header/groups preamble via printf/heredoc.)

After editing rates: `bash scripts/generate-pricing-rules.sh --write && curl -X POST http://localhost:9090/-/reload`.

## `configs/prometheus/rules/cost.yaml` (hand-written)

```yaml
groups:
  - name: shepherd_computed_cost
    interval: 1m
    rules:
      # Cumulative computed cost (counter-shaped; use increase() over it in dashboards/alerts)
      - record: shepherd:claude:computed_cost_usd
        expr: |
          sum by (git_repo, model) (
            shepherd_session_tokens_total{source="claude-code"}
            * on (model, type) group_left ()
            shepherd:model_price:usd_per_mtok
          ) / 1e6

      - record: shepherd:claude:skill_cost_usd
        expr: |
          sum by (git_repo, skill_name, skill_type, model) (
            shepherd_skill_tokens_total
            * on (model, type) group_left ()
            shepherd:model_price:usd_per_mtok
          ) / 1e6

      # What prompt caching saved: cacheRead tokens billed at cacheRead rate instead of input rate
      - record: shepherd:claude:cache_savings_usd
        expr: |
          sum by (git_repo, model) (
            shepherd_session_tokens_total{type="cacheRead"}
            * on (model) group_left ()
            (   max by (model) (shepherd:model_price:usd_per_mtok{type="input"})
              - max by (model) (shepherd:model_price:usd_per_mtok{type="cacheRead"}) )
          ) / 1e6

      # Guardrail: models emitting tokens but absent from the pricing table (silent $0 otherwise)
      - record: shepherd:claude:unpriced_models
        expr: |
          count by (model) (shepherd_session_tokens_total)
          unless
          count by (model) (shepherd:model_price:usd_per_mtok)
```

Naming follows the existing `shepherd:codex:*` colon convention
(`configs/loki/rules/fake/codex.yaml`).

Known accepted limitation: the recorded cost series is gauge-shaped over a counter; `increase()`
handles resets numerically, but a collector restart resets the underlying counters and can
slightly misestimate across the reset. Fine locally.

## Verification

```bash
jq -e . configs/pricing/model-prices.json
bash scripts/generate-pricing-rules.sh | head -20
diff <(bash scripts/generate-pricing-rules.sh) configs/prometheus/rules/pricing-generated.yaml
promtool check rules configs/prometheus/rules/*.yaml          # if promtool installed
docker compose restart prometheus                              # or: curl -X POST localhost:9090/-/reload
curl -s http://localhost:9090/api/v1/rules | jq '[.data.groups[].name]'
# expect: includes "shepherd_model_prices" and "shepherd_computed_cost"
curl -s 'http://localhost:9090/api/v1/query?query=shepherd:model_price:usd_per_mtok' | jq '.data.result | length'
# expect: 4 × number of models
curl -s 'http://localhost:9090/api/v1/query?query=shepherd:claude:computed_cost_usd' | jq
# expect: nonzero once spec 02 data flows
curl -s 'http://localhost:9090/api/v1/query?query=shepherd:claude:unpriced_models' | jq '.data.result'
# expect: [] (empty) once all observed models are priced
```

Sanity: `sum(shepherd:claude:computed_cost_usd)` should land within ~±20% of
`sum(shepherd_claude_code_cost_usage_USD_total)` over the same window (native is Claude's own
list-price estimate; drift beyond that means a wrong rate, a wrong model ID, or broken cache-rate
assumptions).
